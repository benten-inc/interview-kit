# raw ファイル仕様

## 目的

どんなインタビュー形式であっても、質問と回答の原文がそのまま残る状態を作る。
分析・構造化は別工程で行う。raw は原本であり、加工しない。

## 対象となるインタビュー形式

| 形式 | 原文の取得方法 |
|------|---------------|
| Claude Code（テキスト入力） | 回答を受け取るたびにファイルに追記 |
| Claude Code（音声入力） | 同上（音声→テキスト変換後の入力が記録される） |
| 人間が聞いて録音 | 文字起こし後、Q/A に分離して記録 |

## ファイルパス

```
schemas/v0.1/examples/{name}/raw/{YYYY-MM-DD}.md
```

- `{name}`: 対象者名（ローマ字小文字）
- `{YYYY-MM-DD}`: インタビュー実施日
- 同日に複数セッションがある場合: `{YYYY-MM-DD}-2.md`, `{YYYY-MM-DD}-3.md`

## フォーマット

```markdown
---
person: {name}
date: {YYYY-MM-DD}
interviewer: {claude | 人名}
source: {text | voice}
---

Q: {質問文}

A: {回答の原文そのまま}

Q: {質問文}

A: {回答の原文そのまま}

...
```

### ルール

- **A（回答）は一切編集しない。** フィラー（「えっと」「まー」等）、言い直し、脱線もそのまま残す
- **Q（質問）も実際に発した/表示した文をそのまま記録する。** テンプレートの質問文ではなく、実際の質問
- 1つの Q に対して 1つの A。回答の途中で追加質問を挟んだ場合は、そこで Q/A を分ける
- 空行で Q と A を区切る

### frontmatter

| フィールド | 必須 | 値 |
|-----------|------|-----|
| person | yes | 対象者名（ローマ字小文字） |
| date | yes | YYYY-MM-DD |
| interviewer | yes | `claude` または人名（ローマ字小文字） |
| source | yes | `text`（テキスト入力）/ `voice`（録音→文字起こし） |

## Claude Code での書き出し

interviewer スキル実行時、回答を受け取るたびに raw ファイルへ追記する。

```
1. 質問を出す
2. ユーザーが回答する
3. Q: と A: を raw ファイルに追記
4. 構造化分析（種別判定、coverage 更新、次の問い決定）
5. 次の質問を出す → 2 に戻る
```

構造化分析の結果は別ファイル（structured/）に記録する。raw ファイルには原文のみ。

## 録音→文字起こしの場合

1. 録音ファイルを文字起こしする（ツールは問わない）
2. 文字起こしテキストから質問と回答を分離する
3. 上記フォーマットに整形して raw ファイルとして保存する

文字起こし元の録音ファイルは raw ファイルとは別に保管する。
