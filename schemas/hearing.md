# ヒアリング

## 1. コア領域と補助レンズ

### コア領域（トリセツに直接書かれる）

状態マップが confidence / depth を追跡する主領域。外部化レビューのトリガー判定もこの領域で行う。

| ID  | 領域                  | 見たいこと                   | 聞き方の例                                            |
| --- | ------------------- | ----------------------- | ------------------------------------------------ |
| D   | drives              | 何を報酬として感じるか、何がエンジンになるか  | 「報われた」と感じた瞬間。誰にも言われなくても追いかけてしまうもの。気が済まないこと       |
| G   | guardrails          | 何を危険と感じ、何を守るために何を切るか    | 何を否定されると一番きつか。急に身構える場面。追い込まれた時に何を後回しにするか         |
| N   | self_narrative      | どんな自己像を守っているか           | どう思われたいか。絶対に自分はそうではないと思うもの。周りからどう見られると一番しっくりくるか  |
| C   | communication_style | 情報をどう処理・伝達・受容するか        | 考えがまとまる前に話し始めるか、考えてから話すか。大事なことはテキストと口頭どちらが入りやすいか |
| E   | energy_management   | どういうリズムで動き、何で充電し、何で壊れるか | 1日の中で一番冴えている時間帯。人以外で意外と消耗するもの。限界の前兆              |
| S   | contextual_shift    | どの文脈でモードが切り替わるか         | どの人といるとき一番自然か、どの場で硬くなるか                          |

### 補助レンズ（コア領域の別角度）

独立した領域として confidence を追わない。コア領域を聞く時の切り口として使う。

| レンズ | 切り口 | 効くコア領域 |
|------|------|------------|
| 外向 / 内向 | 人と会った後どう過ごしたくなるか。人前で機能する時、自然か頑張っているか | communication_style + energy_management |
| 抽象 / 具体 | 説明する時、全体から入るか具体から入るか | communication_style |
| リスク反応 | 不確実さが燃料か安定が土台か | drives + guardrails |
| 感情反応 | 否定された時どれくらい引きずるか、切り替えは早いか | guardrails |
| 学び方 | 新しいことを始める時まず何をするか | communication_style + drives |
| 信頼の構造 | 何で信頼し、何で壊れ、戻るか | drives + guardrails |
| 飽き・離脱 | 続かなかったものの共通点、不思議と続いているもの | drives |
| 価値観トレードオフ | 正しさと調和、効率と人の気持ちが衝突した場面 | drives + guardrails + self_narrative |

補助レンズは、コア領域が confidence: medium の時に別角度から取り直すために使う。
単独で「外向/内向の領域が confidence: low」のようには追跡しない。

## 2. 判断ルブリックと信頼レベル

### 判断ルブリック: what（何を聞くか）

セッション冒頭で状態マップを見て、以下の優先順位で判断する。

| 優先度 | 条件 | やること |
|--------|------|---------|
| 1 | unresolved_contradictions に priority: high がある | 例外質問 or 別角度の場面で矛盾を浮かせる |
| 2 | confidence: low の領域がある | その領域に触れる場面を聞く |
| 3 | confidence: medium の領域がある | 別文脈の Episode を取る / 有事で確認する |
| 4 | 上記すべてが一定水準 | 新しいエピソードを自由に聞く |

### 信頼レベルと質問の深さ

ループ型では「何回目だからこの質問」ではなく、信頼レベルで聞ける深さが決まる。

| trust_level | 聞ける範囲 | 聞けない範囲 |
|-------------|----------|------------|
| low | 出自、仕事の具体、得意なこと | 恐怖、Guardrails の核心、性格の矛盾 |
| medium | 欲望、自己像、意思決定の癖 | 最も深い不安、誰にも言えないこと、矛盾の直接的な浮上 |
| high | 有事の反応、矛盾、限界、死角 | なし（ただし伝え方は常に慎重に） |

trust_level の判定基準:

| レベル | 根拠の例 |
|--------|---------|
| low → medium | 感情を伴うエピソードを自発的に語った。弱さに関する話題を避けなかった |
| medium → high | 前回のセッション後に自分から追加の話をしてきた。「実は…」で始まる話が出た。矛盾を指摘されても防衛が上がらなかった |

### 言語化の質の追跡

きれいに言語化された回答ほど、再確認が必要。

| verbalization_quality | 意味 | 次のアクション |
|----------------------|------|--------------|
| high（きれいにまとまっている） | 本人の言語能力で整えられた可能性 | 別角度から再確認、場面ベースで具体に降ろす |
| medium（部分的に言語化できている） | 一部は構造に近い | 言語化できていない部分を場面で埋める |
| low（言いよどんでいる、修正が多い） | 言葉になる前の構造に近い可能性 | 急かさない。沈黙を許容。言い直しの方向を記録する |

### 進め方の原則

- ラベルを直接聞かない。具体的な場面で聞く
- 「できるか」ではなく「自然にそうなるか」
- 「好きか」ではなく「エネルギーがどう動くか」
- 回答が抽象的なら「最近だとどの場面ですか」で具体に戻す
- 相手の言葉をそのまま事実にしない（Episode と Inference を分離する）
- きれいにまとまった回答には「今の言葉、自分的にしっくり来てますか？」で精度を確認する
- **抽象語は単独で直撃しない**。「失敗って何？」「追い込まれた時は？」のような抽象語は、先に具体場面（「最近、これは失敗だったなって思った場面は？」）を取って、そこから抽象に上げる。抽象語直撃は質問返し・撤回・防衛を招く
- **出自から入るのを固定し、時系列を基本骨格にする**。初回セッションは §1 出自から入る（信頼構築 + 時系列の開始点）。以降はセクション内・セクション間で「聞け出しやすいところから」を優先して流れで拾う
- **1セッションで全領域を埋めない**。ループ型の強みは「今日は触れない」を選べること。何回かに分ける前提で設計する
- **レイヤー（コア vs 応用）で分けて聞かない**。仕事の具体場面から Drives / Guardrails を取るほうが自然で、Episode が豊かに出る
- **質問集は文面そのままではなく、軸を埋め込んで作り変える**。質問集にある問いの多くは「相手・場・状態・時期で答えが変わる」ことが前提（どう思われたい、報われた瞬間、合わない人、最後に何で決める、など）。丸いまま投げると本人は「普遍解を出さなきゃ」という圧で抽象に逃げる。聞き手はその場の文脈に応じて、分岐軸（誰に対して／どんな場面で／平時か有事か／仕事かプライベートか など）を質問の中に最初から織り込む。例: 「人からどう思われたい？」→「**どういう人に**どう思われたい？」、「報われた瞬間は？」→「**どんな場面で**報われたって感じる？」。軸の選び方は直前の流れと本人の語り口から拾う

## 3. 質問集

### 前提

#### このヒアリングで大事なこと

聞き手の役割は、構造を見つけることではない。
本人が鎧を脱いで話せる場を作り、その人の中にあるものが自然に出てくるのを支えること。

- 質問は分析のためのメスではなく、本人が自分を映すための鏡
- 回答には事実（Episode）と本人の解釈（Inference）が混ざっている。分離は会話後に行う
- 聞き手が賢くなるための場ではなく、本人の中にあるものが形になるための場
- 出てきた言葉は近似値であり、その奥にあるものを場面・矛盾・沈黙から取りに行く

出てきたものを後から外部化し、本人が見える形に残す。それがこのヒアリングの仕事。

#### テンプレは台本ではなくカタログ

下の §ヒアリングの構成 にある質問表は **カタログ** であって、上から順に読み上げる台本ではない。単発で出すと失敗しやすい問い（判断基準・信頼観・恐怖の定義など、抽象を直接取りに行くもの）が多い。

基本は **話の流れの中で拾う**:

- 前の回答に次の質問の具体がすでに含まれていないか見る。含まれていれば繋げる（例: 転職の話が場にある時「あの時、最後に何で決めたんですか？」→ 単発で「迷った時、最後に何で決める？」と出すより強い）
- 含まれていなければ、場面を取りに戻る。抽象問いを単独で出す前に、具体エピソードが場にある状態を作る
- **自然に聞けるものは領域を飛び越えて聞いてしまっていい**。セクション順は領域カバレッジの初期ガイドラインであって、進行順ではない

テンプレの表は「この領域を取るための武器庫」。流れに合う武器を取り出して使う。

#### 出自から入る・時系列を基本骨格にする

**初回セッションは §1 出自から入るのを固定にする**。出自を語ることで信頼構築を兼ねられるし、**時系列（過去 → 現在 → 未来）** を全体の骨格として先に置くと、以降の「現在」「未来」の話に解釈軸が生まれる。

**ただし「コア vs 応用」のレイヤーでは分けない**。出自以降は、仕事の場面から Drives / Guardrails / Self Narrative を流れで取る。「まず抽象で取り、次に仕事で確認する」のような層分けはしない。

**セクション内・セクション間では「聞け出しやすいところから」を優先**:

- §2 以降は、開きやすい側（得意・成功・動機）から入り、奥側（素の Drives・自己像・失敗・否定）に降りていく
- 自然に聞けるものは領域を飛び越えて聞いてしまっていい（§テンプレは台本ではなくカタログ 参照）

#### 1セッションで全領域を埋めない

初回で全部やろうとしない。出自 → §2 を厚めに取り、残り領域は次回以降に分散する。**何回かに分ける前提で設計する**。

ループ型の強みは「今日は触れない」を選べること。trust_level が追いついていない領域を無理に取らない。

### §1 出自

目的: 信頼構築と Person_Background の材料取得を兼ねる。
自分の原点を語ることから入る。ここから入ることが相手への敬意になる。
出自だけで30分使っていい。ここから自然にDrives、Guardrailsまで流れることが多い。

| 質問                                   | 取れるもの                         |
| ------------------------------------ | ----------------------------- |
| どこで生まれて、どんな家庭で育ちましたか？                | 家族構成、環境、Background の直接材料      |
| 親のこと、好きでしたか？                         | 親との関係の質。Drives/Guardrails の原型 |
| 親の価値観で一番影響を受けたものは？                   | 言語化された価値観。Self Narrative の種   |
| 家の中で、言葉にはされてなかったけど「これはダメ」みたいな空気はあった？ | 暗黙のルール。Guardrails の原型         |
| 子どもの頃、親や先生からよく言われたことは？               | 反復フィードバック①。構造が幼少期から外に漏れていた部分  |
| 兄弟や周囲と一番違ったところは？                     | 対比で本人の特異性が浮かぶ                 |

**フォローアップ：**
- 抽象的な回答が出たら「最近だとどの場面？」で具体に降ろす
- 特徴的なエピソードが出たら「それって今思えば特徴的だった？当時は当たり前だった？」

### §2 仕事と、その奥にあるもの

目的: 得意な仕事を語る時、人は最も無防備になる。そこから入って、その奥にある駆動力・報酬・恐怖・自己像まで、流れの中で取る。「コア層を抽象で取り、応用で確認する」レイヤー分けはしない。

聞く順番の目安: **開きやすい側**（得意・成功・動機）から入り、**奥側**（素の Drives・自己像・失敗・否定・不確実）に降りていく。固定ではない。流れが別方向に行けば追う。

| 質問                                                  | 取れるもの                         |
| --------------------------------------------------- | ----------------------------- |
| 仕事の中で一番得意だなって思うことは？                                 | Drivesの仕事面。どこに自己同一性を置いているか    |
| そのコツを人に教えるとしたら、どう言う？                                | 教え方にOSが出る。抽象型/具体型/対人感覚型       |
| なぜそれが得意なんだと思う？                                      | Background/Experienceとのつながり   |
| 今まで一番うまくいったプロジェクトは？                                 | 成功体験の構造。何を見て、何をして、何が起きたか      |
| その中で一番効いた判断は？                                       | 意思決定OSの実例                     |
| 仕事で大切にしていることは？そう思うようになったきっかけは？                      | 仕事上の価値観とExperienceの接続         |
| 今やっていることって、どうやって始まった？なぜこのテーマ？                       | 選択の動機。自己投影の有無                 |
| 最近、「報われたな」って思った瞬間ありますか？ → その中で一番強く感じたのはどれ？          | 報酬の定義（具体→抽象）                  |
| なんか気が済まないことってあります？小さなことでも                           | Drivesの行動レベル                  |
| 無意識に時間を使ってしまっていることって、何ですか？                          | 素のDrives。無意識に引き寄せられるもの        |
| 自分でも「なんでこれやってるんだろう」って思うことは？                         | 無意識の行動パターン                    |
| 人からどう思われたいですか？                                      | 承認ドライバー。Self Narrative        |
| 今やっていることの中に、自分の人生と重なるものはある？                         | 自己投影の自覚度                      |
| これやってなかったら、今何してた？                                   | 仕事と自己同一性の距離                   |
| この能力が裏目に出たことは？                                      | 強みの裏面。認知ギャップの種                |
| 最近、これは失敗だったなって思った場面ありますか？ → それを「失敗」にしてるものって何だと思います？ | 恐怖の定義（具体→抽象）。Guardrailsが凝縮される |
| 最近、否定されて一番きつかった場面ってあります？ → その時、自分ってどうなりました？         | 否定への反応パターン（具体→抽象）             |
| 先が全然読めない状況に放り込まれたら、最初に何する？                          | 不確実場面での初動。リスクの取り方             |

**フォローアップ（なぜを3回聞く）:**
- 「それって、なんでなんですかね？」→ なぜ①。表層理由
- 「その奥にあるものって？」→ なぜ②。本音
- 抽象的な回答が出たら「最近だとどの場面？」で具体に降ろす

### §3 反復フィードバック

目的: 他者からの反復的なフィードバックを複数時点で取り、変化と不変を比較する。

子ども時代の「よく言われたこと」は §1 出自で取得済み。ここでは現在・仕事の2時点を取り、§1 の回答と合わせて3時点で照合する。

| 時点 | 質問 |
|------|------|
| 現在 | 周りの人からよく言われることは？ |
| 仕事 | 一緒に仕事してる人からよく言われることは？ |

各回答に対して必ず聞く：
- 「当たってると思う？」
- 「それ言われた時、どう感じる？」
- 「自分ではそう思ってない部分はある？」

§1 で取った「子どもの頃言われたこと」と合わせて3時点で比較:

- 3時点で変わらないもの = 構造が外に漏れ続けている部分
- 「当たってるけど嫌」= Guardrails
- 「当たってない」= 認知ギャップの入口

### §4 意思決定と対人

目的: 意思決定のOSと、制約下で判断が歪むパターンを取る。認知ギャップはここに集中する。

#### 意思決定のOS（3方向から）

| 質問                             | 角度         |
| ------------------------------ | ---------- |
| 迷った時、最後に何で決める？                 | 判断基準       |
| 後悔してる判断はある？なぜそうなった？            | 失敗パターン     |
| 自分の判断の癖はある？「また同じことやってるな」って思うこと | 自覚しているパターン |

3つの答えが一貫→パターン確定。矛盾→OSの盲点。

#### 制約下の判断

| 質問 | 取れるもの |
|------|----------|
| 判断が何かに引っ張られてるなと感じることは？ | constraint_pull |
| もしそのしがらみがなかったら、本当はどう判断したかった？ | without_constraint → Drivesの本質 |
| この種の「引っ張られ」で後悔したことは？ | regret_pattern |
| やるやらないの判断基準は？ | go_nogo_criteria |
| 今やってることって、やった分が積み上がってる感じありますか？最近どうです？ | compounding |

#### 人間関係と圧

| 質問 | 取れるもの |
|------|----------|
| 人に嫌われるのは平気？ | Guardrailsの核心 |
| 人と仲良くなる時、どういう入り方する？ | 対人パターン |
| 合わない人ってどんなタイプ？ | Guardrailsの裏返し |
| 最近、「伝わってなかったな」って思った場面は？ | 認知ギャップの自覚 |
| その時、本当はどうしたかった？ | 意図と行動のズレ |
| 一番信頼してる人は？なんでその人？ | 信頼の定義にDrives/Guardrailsが凝縮 |
| 最近、プレッシャーが重なった場面ってあります？ → その時、一番最初に動いたのって何でした？（考え？行動？誰かに連絡？） | 有事の初動パターン（具体→抽象） |
| 人にイラっとするのってどんな時？ | 怒りのトリガー。守っているものが出る |
| それって自分でコントロールできてる？ | 自覚度 |
| 夜中に目が覚めて考えてしまうことは？ | 最も深い不安と欲望 |
| 誰にも言えないことはある？ | 孤独の質 |

### §5 トリセツの実用面

目的: 「この人を理解する」だけでなく「この人とどう付き合えばいいか」を書けるようにする。
ここまでのセクションで間接的に取れているものもあるが、ここで明示的に確認・補完する。

#### コミュニケーションの型

| 質問 | 取れるもの |
|------|----------|
| さっきまでの会話で、自分が考えながら話してた瞬間と、考えてから出してた瞬間、両方あったと思います？ | 処理の型。話しながら整理するか、整理してから出すか |
| 大事なことって、口頭・テキスト・対面・電話とかで入ってき方違うと思うんですけど、どれが一番正確に入ってきます？ | 入力の形式 |
| 最近、誰かにフィードバックもらって「これはすっと入ってきたな」って場面ありました？ | フィードバックの受け方 |
| 会議中に黙っている時って、何が起きてますか？ | 沈黙の意味。考え中か、不満か、同意か |
| その場で答えるのと、話しながら整理するのと、持ち帰るのと、状況によると思うんですけど、どれが多いですか？ | 処理の速度と質のバランス |

#### エネルギーのリズム

| 質問 | 取れるもの |
|------|----------|
| 1日の中で一番頭が冴えてる時間帯は？ | エネルギーのピーク |
| 何をすると一番回復する？逆に、休んでるつもりなのに疲れが取れないことは？ | 充電方法と偽の休息 |
| 人と会うこと以外で、意外と消耗するものは？ | 隠れた消耗源（意思決定疲れ、曖昧さ、待ち） |
| 自分が限界に近づいてる時、最初に出るサインは？ | 限界の前兆 |
| 壊れた経験はある？その前に何が起きてた？ | 不可逆な消耗のパターン |

#### 学び方

| 質問 | 取れるもの |
|------|----------|
| 新しいことを始める時、まず何をする？ | 学びの入口（触る、調べる、人に聞く） |
| 新しいこと始める時、全体像が見えるまで動けない派？とりあえず手を動かしながら掴む派？その中間？ | 全体像 vs 手触り |
| 何かを学ぶ時、人に教わる・自分で試す・本を読む・観察する、いろいろあると思うんですけど、最近新しく覚えたことってどうやって入りました？ | 学びのスタイル |
| 学ぶ気がなくなる瞬間ってどんな時？ | 学びが止まる条件 |

#### 信頼の構造

| 質問 | 取れるもの |
|------|----------|
| 人を信頼する時、何を一番見てる？ | 信頼の入口（能力、誠実さ、一貫性、共感） |
| 一発で信頼がなくなるのはどんな時？ | 信頼が壊れるトリガー |
| 一度信頼を失った相手と、また信頼関係を作り直せる方？ | 回復の可能性 |
| 信頼してる人にだけ見せる自分はある？ | 信頼と開示の関係 |

#### 飽き・離脱のパターン

| 質問 | 取れるもの |
|------|----------|
| これまで続かなかったものに共通点はある？ | ガス欠の条件 |
| 飽きる直前って、どんな感覚になる？ | 離脱のサイン |
| 逆に、自分でも不思議なくらい続いてるものは？なぜだと思う？ | 止まらないエンジンの構造 |
| やめて後悔したものはある？ | Drivesとの接続 |

#### 価値観のトレードオフ

| 質問                                 | 取れるもの     |
| ---------------------------------- | --------- |
| 正しいことを言うか、場の空気を守るか、迷った場面は？その時どうした？ | 正しさ vs 調和 |
| 効率を取るか、人の気持ちを取るかで迷った場面は？最近だと？      | 効率 vs 関係性 |
| 自由を取るか、安定を取るかで大きく迷った場面は？           | 自由 vs 安定  |
| その選択の後、自分の中に残ったものは？                | 犠牲の自覚     |

### §6 対比構造（終盤）

目的: ヒアリング中に出てきた矛盾を本人に返す。

聞き方：
- 「さっき〇〇と言ってましたけど、△△とも言ってましたよね。自分の中ではどうつながってますか？」

各矛盾に対して：
- approach_drive（近づきたかった部分）と avoidance_drive（避けたかった部分）を分離する
- which_won（最終的にどちらが行動を決めたか）を確認する
- aware_of_both（本人は両方を自覚していたか）を確認する

#### 締めについて

セッションの締めに「意外だったこと」「一番怖いもの」「知っておいてほしいこと」のような**メタ内省を丸投げする質問は置かない**。過去セッションで複数人が答えに詰まり、場を冷ます構造になることが観測されている。

代わりに:

- **自然に話が語り切られたタイミングで終える**（無理に締め質問を入れない）
- 聞き手が「今日はここまでで」と時間的に締める
- 必要ならメタ評価系で場に返す: 「今日話してみて、一番エネルギー使った問い、どれでした？」「今日の会話で、しっくり来た話としっくり来なかった話、なんとなくあります？」
- 「10年後どうなっていたい？」のような長期ビジョンを取りたい時は、締めではなく Drives セクションの文脈で聞く

メタ内省系の問い（「意外だった？」「知っておいてほしいことは？」）は**聞き手の都合による締め**であり、本人が内省を強制される構造を持つ。トリセツは本人の中から自然に出てくる材料で育てる。

## 4. 構造的に回収するためのテクニック

### 同じOSを3方向から殴る

1つの特性を1つの質問で取ろうとしない。別角度から同じものを聞き、一貫性を確認する。

3つの答えが同じOSを指していれば、それはパターン。矛盾していれば、そこに本質がある。

### なぜを3回聞く

1回目で出るのは表層理由。本音は2回目、無意識は3回目。

- 「なんでですか？」→ 用意された理由
- 「それってなんでなんですかね？」→ 少し考えた本音
- 「その奥に何がありますかね？」→ 本人も初めて言語化する層

### 時系列で崩す

同じテーマを過去→現在→未来で聞く。一貫していれば構造。変化していれば何が変えたかが Experience。

### 「よく言われること」を3時点で聞く

子ども時代 → 現在 → 仕事の文脈で「よく言われること」を聞く。
変わらないもの = 本人の構造が外に漏れ続けている部分。変わったもの = Experience による変化。

### 再訪テクニック（ループ型ヒアリングの核）

同じ領域を別セッションで再訪する時の聞き方。矛盾を指摘するのではなく、自然に浮かせる。

**前回の続きから入る：**
- 「あの話の後、何か変わったことありましたか？」
- 「前回話してくれた〇〇のこと、その後どうなりましたか？」

**別文脈で同じ構造を取る：**
- 前回仕事の文脈で聞いた → 今回プライベートの文脈で聞く
- 前回平時の話を聞いた → 今回有事の場面を聞く

**例外を聞く：**
- 「〇〇を大事にしてるって話だったんですが、それが通せなかった場面ってありますか？」
- 「その時、自分の中でどう折り合いをつけました？」

**Self Narrative の裏付けを取る：**
- 「それを裏付ける最近のエピソード、いくつか思いつきますか？」
- 出てこない → 自己像だけど行動に出ていない可能性
- 出てきたけど全部同じ文脈 → 限定的にしか発揮されていない

**選ばなかった方を聞く：**
- 「その時、切った方は痛かった？」
- 「もう一回同じ場面が来たら、同じ選択する？」
- 「切った方のことを、後から考えることはある？」

**言語化の精度を確認する：**
- 「今の言葉、自分的にしっくり来てますか？」
- 「もうちょっと正確に言うとしたら？」

## 5. 他者セッションの設計

### いつ他者に聞くか

他者ヒアリングは独立したフェーズではない。
状態マップの open_questions に `next: other` が出た時に走る。

#### next: other になる問い

| 問いの性質 | 例 |
|-----------|-----|
| 本人の行動が他者にどう見えているか | 「率直さが威圧に見えているか」 |
| Self Narrative の外部検証 | 「"人を大事にしている" は周囲からも見えているか」 |
| 認知ギャップの特定 | 「本人の沈黙をどう受け取っているか」 |
| 本人が語れない死角 | 「本人が気づいていない影響は何か」 |

### 他者セッションの構造

```
1. 自由印象（5-10分）
   - 汎用的に語ってもらう。予想外の材料が出る場所
   - 「〇〇さんと仕事していて、どんな印象ですか？」

2. 焦点化（本体）
   - 状態マップの open_question に沿って具体場面を聞く
   - 「〇〇さんに大事にされてるなって感じた場面と、
     逆にちょっと冷たいなって感じた場面、両方あれば聞かせてもらえますか？」

3. 例外確認
   - 「普段の印象と違った場面はありましたか？」
   - 「振り返ると、自分が誤解していたかもしれないと思う場面は？」
```

### 聞く順番

1. 何が起きたか
2. どう受け取ったか
3. どう感じたか
4. その受け取りはどの場面から来ているか
5. 例外はあったか

### 取るもの

他者セッションで取れるのは、**他者視点の Episode**（`speaker: 他者, about: 本人`）と、そこから生まれる **Observer 起源の Inference**。別エンティティではなく、通常の Episode / Inference スキーマに収める。

| 項目 | 意味 |
|-----|------|
| Episode.speaker | 発話者（他者） |
| Episode.about | この Episode が反映される人物（本人） |
| Episode.what_happened | 何が起きたか（他者から見える場面） |
| Episode.observer_reaction | 他者がどう受け取ったか |
| Episode.context | 場の属性（power / setting / pressure） |
| Inference.source | `observer`（他者視点由来であることを明示） |
| Inference.bias_risk | 関係や感情による歪みの可能性 |
| Inference.triggering_question | 状態マップのどの問いに対応するか |

### 原則

欲しいのは性格評価ではなく、**その人がどう見えたか、その印象がどの場面から来ているか**。
他者視点の Episode を取る時も、印象の言語化より場面の具体化を優先する。

## 6. 認知ギャップの取り方

### 認知ギャップとは何か

単なる「誤解」ではない。**同じ事象に対して、異なる次元で解釈している状態**。
どちらかが間違っているのではなく、処理している次元が違う。

ズレが生まれる次元は4つ：

| 次元 | 定義 | 例 |
|------|------|-----|
| **価値観** | 何を大事にしているかが違う | 本人は事実/効率を優先 → 相手は関係性/安全を優先 |
| **抽象度** | どの粒度で処理しているかが違う | 本人は構造の話 → 相手は「来月何するの？」 |
| **前提** | 当たり前にしている暗黙知が共有されていない | 本人は「リソースは作るもの」→ 相手は「有限な制約」 |
| **視点の高さ** | 見ている時間軸・スコープが違う | 本人は3年後を見ている → 相手は今期を見ている |

### 認知ギャップのパターン

各パターンに4次元のどれが主因かを併記。

| パターン | 本人の認識 | 相手の受け取り | 主な次元 |
|---------|----------|-------------|---------|
| 威圧 / 率直 | 正しいことを早く言った | 攻撃された | 価値観（事実 vs 安全） |
| 閉じる / 拒絶 | 考えている | 怒っている、拒絶 | 価値観（思考 vs 感情） |
| 任せる / 放置 | 信頼して任せた | 見捨てられた | 価値観（成長 vs 安全） |
| 基準 / 冷たさ | 組織として正しい判断 | 人を見ていない | 価値観（構造 vs 承認） |
| 外向skill / 内向OS | この場では機能した | この人はこういう人だ | 価値観（機能 vs 本質） |
| 有事 / 平時 | 追い込まれて反応が変わった | この人の本性が出た | 価値観（状況適応 vs 一貫性） |
| 事業選択 / 自己投影 | 合理的に選んだ | 個人の物語が乗りすぎ | 前提（本人も無自覚な動機） |
| 意思決定の速さ | 速く決めた | 粗い、恣意的 | 前提 + 抽象度 |
| 方針変更 / ブレ | 学習して更新した | 一貫性がない | 視点の高さ + 前提 |
| リスク / 無謀 | 勝ち筋を見て張った | 無謀 | 視点の高さ + 前提 |
| 長期投資 / 遅さ | 長期の構造を見ている | 今を見ていない | 視点の高さ |
| 評価 / 冷たさ | 実力で評価した | 人を見ていない | 価値観 + 抽象度 |

### 聞き方（4次元共通）

**価値観のズレを取る：**
- 「その時、あなたが一番大事にしてたことは？」
- 「相手が大事にしていたものって、同じだった？」

**抽象度のズレを取る：**
- 「その話、相手にはどこまで伝わっていたと思う？」
- 「自分が見えていて、相手に見えていなかったものは？」

**前提のズレを取る：**
- 「それって、言わなくても伝わると思ってた？」
- 「あなたの当たり前で、相手の当たり前じゃなかったものは？」

**視点の高さのズレを取る：**
- 「その判断、何年先を見てた？」
- 「相手はどれくらい先を見てたと思う？」

### 優先して聞く場面

- 部下や共同創業者とのズレ
- 採用・初対面での印象差
- 投資家・顧客との誤解
- 場によって印象が変わった場面
- 圧がかかって判断や言動が変わった場面
- 任せたつもりが伝わらなかった場面
- 正しさを優先して関係に歪みが出た場面

## 7. 矛盾の扱い方

矛盾を指摘すると防衛が上がり、場が壊れる。
矛盾は「指摘する」のではなく「自然に浮かぶ場面を作る」。

### テクニック

**両方を肯定して並べる：**
> 「〇〇の話も分かるし、△△の話もすごく分かるんですよ。これって自分の中ではどういう感じですか？」

矛盾を解消させようとしない。両方並べて、本人に関係を語らせる。

**時間差で同じ場所に着地させる：**

別の話題から入って、結果的に同じ場所に着くように聞く。
聞き手が矛盾を言語化しない。本人の口から出てくるように、場面を選ぶ。

**例外を聞く：**
> 「〇〇を大事にしてるって話だったんですが、それが通せなかった場面ってありますか？」

矛盾の指摘ではなく「例外の確認」。出てくるエピソードは Self Narrative との乖離そのもの。

**第三者の声を借りる：**
> 「周囲の人から『あの人は〇〇だ』って声があったりするんですが、自分的にはどうですか？」

攻撃の主体を聞き手から消す。

## 8. 聞き手の振る舞い

### 根本姿勢

この人の中にあるものは、今出てきた言葉より必ず豊かで複雑なはずだ。
言葉は入口であって終点ではない。
これは不信ではなく敬意。

### 場内 vs 場外で言語の扱いが違う

聞き手の仕事は「本人の言語能力の構造」ではなく「本人の構造」を取りに行くこと。そのためにスコープを分ける。

| スコープ | 言語ポリシー |
|---------|------------|
| 場の中（ヒアリング中） | **本人の言葉を上書きしない / 解釈を差し込まない**。聞き手は問いで空白を作り、本人が自分の言葉で奥に進めるようにする |
| 場の外（セッション後分析 / Pattern 構築） | **聞き手が本人を超えて言語化してよい**。場面・身体・矛盾・沈黙から見えた構造を、本人の入口の言葉に縛らず記述する。ただし Episode 参照は必ず持つ |
| 外部化レビュー | **本人の言い直しを優先する**。成果物を本人に読ませ、違和感があれば本人の言葉で書き直す |

詳細は `concept.md` §Person_Pattern §言語の責任分担 を参照。

### やること

- 出てきた言葉を「入口」として扱い、その奥に進む
- 「今の言葉、自分的にしっくり来てますか？」で言語化の精度を本人に確認させる
- 言いよどみ、修正、沈黙を回答より重視する
- きれいにまとまった回答ほど、別角度からもう一度触る
- 相手の言葉をそのまま繰り返してから次の問いに入る
- 「なぜ」を3回聞く。表層で止めない
- 同じOSを複数の角度から確認する
- 沈黙を許容する。考え込んだ後に出てくる言葉を待つ
- 想定外の方向に話が行っても追う。本人が行きたい方向にエピソードがある
- 各セッション後にカバレッジを確認し、空白を埋めに行く
- 回答に含まれる Inference（本人の解釈）と Episode（事実）を聞き分ける

### やらないこと

- 「なるほど」「面白い」で評価しない。「もう少し聞いていいですか」で続ける
- 二択で回答を狭めない
- 聞き手が期待する方向を透かさない
- 数を指定しない（「3つ挙げて」は重みの順序を壊す）
- 深い質問を信頼構築の前にしない
- 気づかせようとしない（気づきは結果であって目的ではない）
- 本人の Inference をそのまま事実として扱わない
- 聞き手の仮説は会話中に出さない。記録時に仮説メモとして残す
- 解釈を先に差し込まない。「それって〇〇みたいな？」は聞き手の結論が先に出て、本人の言葉が奪われる。「何がそうさせたと思う？」と空白で聞く
- 「それは違うと思います」と本人の言葉を否定しない
- 本人の言語化を聞き手が上書きしない
- 言い直しを急かさない

## カバレッジ確認

各セクション後に、取りたい領域の地図と照合する。

自然に出た領域と、まだ空白の領域を整理する。
空白の領域には、場面ベースの問いで触りに行く。
ただし、項目名は出さない。

## 記録のルール

### 仮説メモの形式

- 直接の結論ではなく仮説として記録する
- 後で検証したいポイントを明記する
- 本人が繰り返し語ったキーワードを残す
- 明らかに強い反応が出たテーマを記録する
- Inference（本人の解釈）と Episode（事実）を分離して記録する

### マッピング

会話後、各回答を「取りたい領域」にマッピングして仮説メモを作る。

- 1つのエピソードが複数の領域にマッピングされることは普通
- マッピングは会話後に行う。会話中は意識しない
- 確度（エピソード数、一貫性）を必ずつける
