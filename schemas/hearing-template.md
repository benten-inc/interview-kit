# ヒアリングテンプレート

その人のトリセツを育てるための、具体的な質問集。

取りたい領域の一覧は `hearing-design.md` を参照。
このファイルは「どう聞くか」に集中する。

各質問には `trust_required`（この質問を聞くのに必要な信頼レベル）を付記する。
ループ型のヒアリングでは「何回目だからこの質問」ではなく、信頼レベルと状態マップが使う質問を決める。

---

## 前提

### このヒアリングで大事なこと

聞き手の役割は、構造を見つけることではない。
本人が鎧を脱いで話せる場を作り、その人の中にあるものが自然に出てくるのを支えること。

- 質問は分析のためのメスではなく、本人が自分を映すための鏡
- 回答には事実（Episode）と本人の解釈（Inference）が混ざっている。分離は会話後に行う
- 聞き手が賢くなるための場ではなく、本人の中にあるものが形になるための場
- 出てきた言葉は近似値であり、その奥にあるものを場面・身体・矛盾・沈黙から取りに行く

出てきたものを後から外部化し、本人が見える形に残す。それがこのヒアリングの仕事。

---

## セッション冒頭のプロトコル（毎回固定）

初回以外のすべての本人セッションは、以下から始める。

| 質問 | 取れるもの |
|------|----------|
| 前回の後、考えたこととか気になったことってありましたか？ | セッション間の内省。最も信頼できるデータ |
| それについて、もう少し聞いていいですか？ | 内省の深掘り |

何も考えなかった場合、それ自体が材料。前回のテーマがその人にとって軽かった可能性。
出てきたものをまず全部聞いてから、判断ルブリックに従って本題に入る。

## 構造的に回収するためのテクニック

### 同じOSを3方向から殴る

1つの特性を1つの質問で取ろうとしない。別角度から同じものを聞き、一貫性を確認する。

3つの答えが同じOSを指していれば、それはパターン。矛盾していれば、そこに本質がある。

### なぜを3回聞く

1回目で出るのは表層理由。本音は2回目、無意識は3回目。

- 「なんでですか？」→ 用意された理由
- 「それってなんでなんですかね？」→ 少し考えた本音
- 「その奥に何がありますかね？」→ 本人も初めて言語化する層

### 時系列で崩す

同じテーマを過去→現在→未来で聞く。一貫していれば構造。変化していれば何が変えたかが Experience。

### 「よく言われること」を3時点で聞く

子ども時代 → 現在 → 仕事の文脈で「よく言われること」を聞く。
変わらないもの = 本人の構造が外に漏れ続けている部分。変わったもの = Experience による変化。

### 再訪テクニック（ループ型ヒアリングの核）

同じ領域を別セッションで再訪する時の聞き方。矛盾を指摘するのではなく、自然に浮かせる。

**前回の続きから入る：**
- 「あの話の後、何か変わったことありましたか？」
- 「前回話してくれた〇〇のこと、その後どうなりましたか？」

**別文脈で同じ構造を取る：**
- 前回仕事の文脈で聞いた → 今回プライベートの文脈で聞く
- 前回平時の話を聞いた → 今回有事の場面を聞く

**例外を聞く：**
- 「〇〇を大事にしてるって話だったんですが、それが通せなかった場面ってありますか？」
- 「その時、自分の中でどう折り合いをつけました？」

**Self Narrative の裏付けを取る：**
- 「それを裏付ける最近のエピソード、いくつか思いつきますか？」
- 出てこない → 自己像だけど行動に出ていない可能性
- 出てきたけど全部同じ文脈 → 限定的にしか発揮されていない

**選ばなかった方を聞く：**
- 「その時、切った方は痛かった？」
- 「もう一回同じ場面が来たら、同じ選択する？」
- 「切った方のことを、後から考えることはある？」

**身体に逃がす：**
- 「その話をしてる時、身体はどんな感じですか？」
- 「さっきの話とこっちの話、どっちの方が身体が楽ですか？」

**言語化の精度を確認する：**
- 「今の言葉、自分的にしっくり来てますか？」
- 「もうちょっと正確に言うとしたら？」

## ヒアリングの構成

### 1. 出自（信頼構築 + Background）

目的: 信頼構築と Person_Background の材料取得を兼ねる。
自分の原点を語ることから入る。ここから入ることが相手への敬意になる。
出自だけで30分使っていい。ここから自然にDrives、Guardrailsまで流れることが多い。

| 質問 | 取れるもの |
|------|----------|
| どこで生まれて、どんな家庭で育ちましたか？ | 家族構成、環境、Background の直接材料 |
| 親のこと、好きでしたか？ | 親との関係の質。Drives/Guardrails の原型 |
| 親の価値観で一番影響を受けたものは？ | 言語化された価値観。Self Narrative の種 |
| 家の中で、言葉にはされてなかったけど「これはダメ」みたいな空気はあった？ | 暗黙のルール。Guardrails の原型 |
| 子どもの頃、親や先生からよく言われたことは？ | 反復フィードバック①。構造が幼少期から外に漏れていた部分 |
| 兄弟や周囲と一番違ったところは？ | 対比で本人の特異性が浮かぶ |
| 子どもの頃と今の自分、根っこは同じだと思う？ | 不変性。何が同じで何が変わったか |
| 親の期待と自分の自由はぶつかったことある？ | 期待と自由の対比構造 |

**フォローアップ：**
- 抽象的な回答が出たら「最近だとどの場面？」で具体に降ろす
- 特徴的なエピソードが出たら「それって今思えば特徴的だった？当時は当たり前だった？」

### 2. 欲望と恐怖（Drives / Self Narrative / Guardrails）

目的: 行動の裏にある駆動力に触れる。「どういう人間か」ではなく「なぜそう動くか」。

| 質問 | 取れるもの |
|------|----------|
| 自分で設計できる時間があったら、何に使う？ | 素のDrives。何に引き寄せられるか |
| それって、なんでなんですかね？ | なぜ①。表層理由 |
| そこの奥にあるものって？ | なぜ②。本音 |
| 自分でも「なんでこれやってるんだろう」って思うことは？ | 無意識の行動パターン |
| 人からどう思われたいですか？ | 承認ドライバー。Self Narrative |
| 逆に、一番嫌な思われ方は？ | 恐怖ドライバー。Guardrails |
| お金と時間が無限にあったら、何する？ | 制約を外した時に残るDrives |
| 「報われた」と一番強く感じた瞬間は？ | 報酬の定義 |
| 「失敗」ってどういう状態？ | 恐怖の定義。Guardrailsが凝縮される |
| 先が全然読めない状況に放り込まれたら、最初に何する？ | 不確実場面での初動。リスクの取り方 |
| 気が済まないことは？ | Drivesの行動レベル |
| 否定された時、自分はどうなる？ | 否定への反応パターン |
| 最近、否定されて一番堪えたのはどんな場面？ | 何に否定されると堪えるかに守っている自己像が出る |
| 身体的に一番いい状態ってどんな時？ | 身体性のDrives |

### 3. 仕事の具体（OSが丸出しになる場所）

目的: 得意な仕事を語る時、人は最も無防備になる。ここでOSの全体像が取れる。

| 質問 | 取れるもの |
|------|----------|
| 仕事の中で一番得意だなって思うことは？ | Drivesの仕事面。どこに自己同一性を置いているか |
| そのコツは？人に教えるとしたら | 教え方にOSが出る。抽象型/具体型/対人感覚型 |
| なぜそれが得意なんだと思う？ | Background/Experienceとのつながり |
| 今まで一番うまくいったプロジェクトは？ | 成功体験の構造。何を見て、何をして、何が起きたか |
| その中で一番効いた判断は？ | 意思決定OSの実例 |
| この能力が裏目に出たことは？ | 強みの裏面。認知ギャップの種 |
| 仕事で大切にしていることは？その原体験は？ | 仕事上の価値観とExperienceの接続 |
| 今やっていることって、どうやって始まった？なぜこのテーマ？ | 選択の動機。自己投影の有無 |
| これやってなかったら、今何してた？ | 仕事と自己同一性の距離 |
| 今やっていることの中に、自分の人生と重なるものはある？ | 自己投影の自覚度 |

### 4. 反復フィードバック（3時点で照合）

目的: 他者からの反復的なフィードバックを3時点で取り、変化と不変を比較する。

| 時点 | 質問 |
|------|------|
| 子ども時代 | 親や先生からよく言われたことは？（セクション1で取得済み） |
| 現在 | 周りの人からよく言われることは？ |
| 仕事 | 一緒に仕事してる人からよく言われることは？ |

各回答に対して必ず聞く：
- 「当たってると思う？」
- 「それ言われた時、どう感じる？」
- 「自分ではそう思ってない部分はある？」

3時点で変わらないもの = 構造が外に漏れ続けている部分。
「当たってるけど嫌」= Guardrails。「当たってない」= 認知ギャップの入口。

### 5. 意思決定と制約（認知ギャップの核心）

目的: 意思決定のOSと、制約下で判断が歪むパターンを取る。認知ギャップはここに集中する。

#### 意思決定のOS（3方向から）

| 質問 | 角度 |
|------|------|
| 迷った時、最後に何で決める？ | 判断基準 |
| 後悔してる判断はある？なぜそうなった？ | 失敗パターン |
| 自分の判断の癖はある？「また同じことやってるな」って思うこと | 自覚しているパターン |

3つの答えが一貫→パターン確定。矛盾→OSの盲点。

#### 制約下の判断

| 質問 | 取れるもの |
|------|----------|
| 判断が何かに引っ張られてるなと感じることは？ | constraint_pull |
| もしその制約がなかったら、本当はどう判断したかった？ | without_constraint → Drivesの本質 |
| この種の「引っ張られ」で後悔したことは？ | regret_pattern |
| やるやらないの判断基準は？ | go_nogo_criteria |
| 積み上げると強みになるか、リセットされるかで判断する？ | compounding |
| これって10年後もやってると思う？ | universality |

#### 人間関係と圧

| 質問 | 取れるもの |
|------|----------|
| 人に嫌われるのは平気？ | Guardrailsの核心 |
| 人と仲良くなる時、どういう入り方する？ | 対人パターン |
| 合わない人ってどんなタイプ？ | Guardrailsの裏返し |
| 最近、「伝わってなかったな」って思った場面は？ | 認知ギャップの自覚 |
| その時、本当はどうしたかった？ | 意図と行動のズレ |
| 一番信頼してる人は？なんでその人？ | 信頼の定義にDrives/Guardrailsが凝縮 |
| 追い込まれた時、最初に出る反応は？ | 有事の初動パターン |
| 人にイラっとするのってどんな時？ | 怒りのトリガー。守っているものが出る |
| それって自分でコントロールできてる？ | 自覚度 |
| 夜中に目が覚めて考えてしまうことは？ | 最も深い不安と欲望 |
| 誰にも言えないことはある？ | 孤独の質 |

### 6. トリセツの実用面（コミュニケーション・エネルギー・学び・信頼・飽き・トレードオフ）

目的: 「この人を理解する」だけでなく「この人とどう付き合えばいいか」を書けるようにする。
ここまでのセクションで間接的に取れているものもあるが、ここで明示的に確認・補完する。

#### コミュニケーションの型

| 質問 | 取れるもの |
|------|----------|
| 考えがまとまる前に話し始める方？それとも考えてから話す方？ | 処理の型。話しながら整理するか、整理してから出すか |
| 大事なことを伝えられる時、口頭とテキスト、どっちの方が正確に受け取れる？ | 入力の形式 |
| フィードバックは直球で言われた方がいい？前置きがあった方が入りやすい？ | フィードバックの受け方 |
| 会議中に黙っている時って、何が起きてる？ | 沈黙の意味。考え中か、不満か、同意か |
| 即答を求められるのと、持ち帰って考えるの、どっちが自然？ | 処理の速度と質のバランス |

#### エネルギーのリズム

| 質問 | 取れるもの |
|------|----------|
| 1日の中で一番頭が冴えてる時間帯は？ | エネルギーのピーク |
| 何をすると一番回復する？逆に、休んでるつもりなのに回復しないことは？ | 充電方法と偽の休息 |
| 人と会うこと以外で、意外と消耗するものは？ | 隠れた消耗源（意思決定疲れ、曖昧さ、待ち） |
| 自分が限界に近づいてる時、最初に出るサインは？ | 限界の前兆 |
| 壊れた経験はある？その前に何が起きてた？ | 不可逆な消耗のパターン |

#### 学び方

| 質問 | 取れるもの |
|------|----------|
| 新しいことを始める時、まず何をする？ | 学びの入口（触る、調べる、人に聞く） |
| 全体像が見えないまま手を動かすのは平気？それとも気持ち悪い？ | 全体像 vs 手触り |
| 人に教わるのと自分で試行錯誤するの、どっちが自然？ | 学びのスタイル |
| 学ぶ気が失せる瞬間ってどんな時？ | 学びが止まる条件 |

#### 信頼の構造

| 質問 | 取れるもの |
|------|----------|
| 人を信頼する時、何を一番見てる？ | 信頼の入口（能力、誠実さ、一貫性、共感） |
| 一発で信頼がなくなるのはどんな時？ | 信頼が壊れるトリガー |
| 一度信頼を失った相手と、また信頼関係を作れる方？ | 回復の可能性 |
| 信頼してる人にだけ見せる自分はある？ | 信頼と開示の関係 |

#### 飽き・離脱のパターン

| 質問 | 取れるもの |
|------|----------|
| これまで続かなかったものに共通点はある？ | ガス欠の条件 |
| 飽きる直前って、どんな感覚になる？ | 離脱のサイン |
| 逆に、自分でも不思議なくらい続いてるものは？なぜだと思う？ | 止まらないエンジンの構造 |
| やめて後悔したものはある？ | Drivesとの接続 |

#### 価値観のトレードオフ

| 質問 | 取れるもの |
|------|----------|
| 正しいことを言うか、場の空気を守るか、迷った場面は？その時どうした？ | 正しさ vs 調和 |
| 効率を取るか、人の気持ちを取るか。最近ぶつかった場面は？ | 効率 vs 関係性 |
| 自由を取るか、安定を取るか。人生で大きく迫られた場面は？ | 自由 vs 安定 |
| その選択の後、自分の中に残ったものは？ | 犠牲の自覚 |

### 7. 対比構造で確認（終盤）

目的: ヒアリング中に出てきた矛盾を本人に返す。

聞き方：
- 「さっき〇〇と言ってましたけど、△△とも言ってましたよね。自分の中ではどうつながってますか？」

各矛盾に対して：
- approach_drive（近づきたかった部分）と avoidance_drive（避けたかった部分）を分離する
- which_won（最終的にどちらが行動を決めたか）を確認する
- aware_of_both（本人は両方を自覚していたか）を確認する

#### 最後の質問

| 質問 | 取れるもの |
|------|----------|
| ここまで話してみて、自分で意外だったことは？ | 本人の気づき |
| 一番怖いものは？ | Guardrailsの核心 |
| 10年後、どうなっていたい？……即答できた？ | 長期Drives。即答できるかどうかが材料 |
| 普段言わないけど、分かってもらうには知っておいてほしいことは？ | 本人が伝えたい情報 |

### 8. フィードバックセッション（別日）

目的: 集めた構造を本人に返す。「お前違ってるよ」が起きる場所。
Validation を通過した Self_Narrative_Divergence がある場合にのみ実施する。

**前提**: 収束条件を満たしていること（状態マップの主要領域が confidence: high、Self_Narrative_Divergence が少なくとも1つ validated、open_questions に priority: high がない、trust_level: high）。

#### ステップ1: 過小評価を返す（防衛を下げる）

| 質問 | 取れるもの |
|------|----------|
| ヒアリングの中で見えてきたんですが、あなたにはこういう強みがありそうです。自分では気づいてましたか？ | 過小評価の自覚度 |
| 周囲はここを頼りにしているようです。意外ですか？ | 他者視点とのズレ |
| この強み、自分で使いこなせてる感覚はある？ | 強みの活用度 |

#### ステップ2: 誤帰属を返す

| 質問 | 取れるもの |
|------|----------|
| この行動の理由、ご自身ではこう説明してくれました。でも別の角度から見ると、こうも読めます。どう思いますか？ | 誤帰属への反応 |
| もしその読みが当たってるとしたら、自分の中で何が変わりますか？ | 受容の深さ |

#### ステップ3: 過大評価を返す（最も慎重に）

| 質問 | 取れるもの |
|------|----------|
| 「自分は〇〇だ」とおっしゃっていました。ただ、いくつかの場面ではこういう行動がありました。自分ではどう見えますか？ | 乖離への反応。エピソードで返す |
| その場面での行動と、自分のイメージ、どちらの方が本当の自分に近いと思いますか？ | 自己像の柔軟性 |
| もし周囲にも同じように見えていたとしたら、どう感じますか？ | 他者視点の受容度 |

#### ステップ4: 死角を浮かせる（直接言わない）

| 質問 | 取れるもの |
|------|----------|
| ここまでの話の中で、自分で意外だったことは？ | 本人の中で浮かび上がった死角 |
| 周囲からこういう声がありました。心当たりはありますか？ | 死角の入口 |
| もし自分に見えていないものが1つあるとしたら、何だと思いますか？ | メタ認知。答えられるかどうか自体が材料 |

#### 記録

- 本人の反応（反発、沈黙、驚き、涙、即答、考え込み）をすべて Episode として記録する
- 受容 / 拒否 / 保留 を `feedback_status` に記録する
- 拒否の場合、何に対して防衛が出たかを記録する（Guardrails の追加材料になる）

## Episodeの取り方

### 基本フォーマット

各 Episode では最低限、以下を取る。

- 何が起きたか
- その時、相手は誰だったか
- 権力差はあったか
- 公の場か、私的な場か
- 平時か、緊急時か
- 自分はどう振る舞ったか
- その時どう感じたか
- 結果として何が起きたか
- その後どう回復したか
- 後から見ると、何を守ろうとしていたと思うか

### 対比構造

- `approach_drive`: 近づきたかった部分（ワクワク、好奇心、獲得欲）
- `avoidance_drive`: 避けたかった部分（不安、恐怖、損失回避）
- `which_won`: 最終的にどちらが行動を決めたか
- `aware_of_both`: 本人は両方を自覚していたか

聞き方：
- 「その場面の中で、ワクワクしてた部分と不安だった部分、両方あった？」
- 「やりたくてやったのか、やらないとまずいからやったのか、どっちが先だった？」
- 「その判断、攻めだった？守りだった？」

### 意思決定の軸

- `constraint_pull`: 判断を歪めている制約
- `who_pulls`: 誰の存在が影響しているか
- `without_constraint`: 制約がなければどう判断したかったか
- `regret_pattern`: この種の歪みで後悔したことはあるか

### 事業選択の構造

- `core_fit`: 自分のコアが活かせるか
- `universality`: 普遍的か、トレンドか
- `go_nogo_criteria`: やる/やらないの判断基準
- `compounding`: 積み上げると強みになるか
- `execution_style`: 進め方のスタイル

### 追加フィールド

- `body_response`: 身体的に何が起きたか
- `repeated_pattern`: この種のことは前にもあったか
- `feedback_from_others`: この場面について周囲から何か言われたか

## 聞き手の振る舞い

### 根本姿勢

この人の中にあるものは、今出てきた言葉より必ず豊かで複雑なはずだ。
言葉は入口であって終点ではない。
これは不信ではなく敬意。

### やること

- 出てきた言葉を「入口」として扱い、その奥に進む
- 「今の言葉、自分的にしっくり来てますか？」で言語化の精度を本人に確認させる
- 言いよどみ、修正、沈黙を回答より重視する
- きれいにまとまった回答ほど、別角度からもう一度触る
- 相手の言葉をそのまま繰り返してから次の問いに入る
- 「なぜ」を3回聞く。表層で止めない
- 同じOSを複数の角度から確認する
- 沈黙を許容する。考え込んだ後に出てくる言葉を待つ
- 想定外の方向に話が行っても追う。本人が行きたい方向にエピソードがある
- 各セッション後にカバレッジを確認し、空白を埋めに行く
- 回答に含まれる Inference（本人の解釈）と Episode（事実）を聞き分ける
- 身体性に注意する。「その時、身体はどうだった？」を入れる

### やらないこと

- 「なるほど」「面白い」で評価しない。「もう少し聞いていいですか」で続ける
- 二択で回答を狭めない
- 聞き手が期待する方向を透かさない
- 数を指定しない（「3つ挙げて」は重みの順序を壊す）
- 深い質問を信頼構築の前にしない
- 気づかせようとしない（気づきは結果であって目的ではない）
- 本人の Inference をそのまま事実として扱わない
- 聞き手の仮説は会話中に出さない。記録時に仮説メモとして残す
- 「それは違うと思います」と本人の言葉を否定しない
- 本人の言語化を聞き手が上書きしない
- 言い直しを急かさない

## カバレッジ確認

各セクション後に、取りたい領域の地図と照合する。

自然に出た領域と、まだ空白の領域を整理する。
空白の領域には、場面ベースの問いで触りに行く。
ただし、項目名は出さない。

## 記録のルール

### 仮説メモの形式

- 直接の結論ではなく仮説として記録する
- 後で検証したいポイントを明記する
- 本人が繰り返し語ったキーワードを残す
- 明らかに強い反応が出たテーマを記録する
- Inference（本人の解釈）と Episode（事実）を分離して記録する

### マッピング

会話後、各回答を「取りたい領域」にマッピングして仮説メモを作る。

- 1つのエピソードが複数の領域にマッピングされることは普通
- マッピングは会話後に行う。会話中は意識しない
- 確度（エピソード数、一貫性）を必ずつける
